# Feature Slice: CloudFactory Notification Webhooks

Status: Done
Date: 2026-07-20
Parent: `docs/rfc/2026-07-16-cloudfactory-partner-integration.md`
Owner: Codex

## Goal

Use CloudFactory notification webhooks to trigger fast, idempotent reconciliation while retaining
scheduled polling as the authoritative safety net.

## User-Visible Behavior

Administrators with a connected Partner Admin account can enable or disable webhook registrations
from Cloud Factory settings and see recent delivery and processing status without seeing the key.

## Scope

Current Notification API event discovery and webhook registration; encrypted random shared key;
`X-API-KEY` validation; partner binding; request-size and rate limits; deterministic
`EventKey + CreatedAt + PartnerGuid` deduplication; queued reconciliation; receipt history; safe
removal before token revocation; and polling fallback.

## Out Of Scope

Cryptographic signatures not offered by CloudFactory, trusting event payloads as domain truth, and
disabling scheduled reconciliation.

## Data Touched

Integration encrypted secrets and settings, `cloudfactory_webhook_receipts`, audit events, public
Integration route, queue jobs, and Cloud Factory webhook registrations.

## Permissions

The public endpoint requires the shared header key and matching partner rather than a Nexum session.
Only Nexum administrators can manage provider registrations. The Portal account requires Partner
Admin.

## Tests

Header rejection, old legitimate retry acceptance, deterministic duplicate handling, minimized
payload storage, registration header contents, provider removal, event-to-sync mapping, queue
processing, migration, route, Blade, and authenticated settings smoke checks.

## Documentation

CloudFactory RFC, webhook ADR, administrator Knowledge, production validation slice, and
`docs/human-review.md`.

## Done Criteria

- [x] The key is generated by Nexum, encrypted, and never rendered.
- [x] The current CloudFactory Notification API registration format is used.
- [x] Identical retries create one receipt and one queued reconciliation.
- [x] Webhook data only triggers the normal authenticated provider sync.
- [x] Disabling and revocation remove registrations before deleting credentials.
- [x] Polling remains enabled.
- [x] Automated verification passes on Dev.
