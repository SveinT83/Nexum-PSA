Task templates are planned as the standard way to create repeatable work.

A template group represents a reusable package, such as onboarding equipment,
preparing a new customer, or performing a recurring maintenance job.

Template items represent the tasks inside that package. They can include:

- nested child tasks
- checklist items
- default queue
- default priority
- default assignee
- estimated time
- category and tags
- dependency rules

Templates should be applied from the record where the work starts, such as a
ticket, asset, or client. The generated tasks keep the owner context so the work
can be found from both the Task list and the owning record.

Recurring tasks use templates. The recurrence record decides when the next task
set should be created, while the template decides what should be created.
